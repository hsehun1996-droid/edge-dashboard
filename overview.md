# Project Overview

## 1. 프로젝트 개요

| 항목 | 내용 |
|---|---|
| **프로젝트명** | (미정) |
| **한 줄 정의** | 국내 주식 투자자를 위한 올인원 투자 정보 대시보드 |
| **목표** | 흩어진 투자 정보(시세·뉴스·수출입·글로벌 지수)를 한 화면에서 빠르게 파악할 수 있도록 한다 |
| **상태** | 기획 / 설계 단계 |

---

## 2. 문제 정의

국내 개인 투자자는 투자 판단을 위해 여러 사이트를 오가야 한다.

- 네이버 증권 → 시세 확인
- 한국무역협회(KITA) → 수출입 데이터 확인
- 인베스팅닷컴 → 글로벌 지수 확인
- 각종 언론사 → 뉴스 수집
- 증권사 HTS → 컨센서스 확인

**이 프로젝트는 위 흐름을 하나의 서비스로 통합한다.**

---

## 3. 타겟 유저

**주요 타겟: 국내 개인 주식 투자자**

| 유저 유형 | 특징 |
|---|---|
| 직장인 투자자 | 시간이 없어 빠른 정보 소비 필요, 모바일 중심 |
| 주식 공부 중인 초보 | 용어·지표 설명이 있어야 이해 가능 |
| 적극적 단타 투자자 | 실시간성과 차트 정확도 중요 |

---

## 4. 핵심 기능

### Phase 1 — MVP (우선 구현)

- [ ] **종목 검색** — 종목명 또는 티커로 검색, 자동완성
- [ ] **종목 상세 페이지** — 현재가, 등락, 거래량, 인터랙티브 차트 (1D/1W/1M/1Y)
- [ ] **컨센서스 정보** — 목표주가, 투자의견(매수/중립/매도) 요약
- [ ] **관련 뉴스** — 종목별 최신 뉴스 피드
- [ ] **국내 증시 홈** — KOSPI / KOSDAQ / KRX 주요 지수 현황

### Phase 2 — 확장

- [ ] **글로벌 시장 현황** — 나스닥, S&P500, 닛케이, 상해종합 등
- [ ] **수출입 데이터** — 품목별·국가별 수출입 추이 (KITA 데이터)
- [ ] **업종/테마 지도** — 섹터별 등락 히트맵
- [ ] **관심 종목 즐겨찾기** — 로그인 후 개인 워치리스트 저장

### Phase 3 — 고도화

- [ ] **AI 요약** — 뉴스·공시 자동 요약 (Claude API 연동)
- [ ] **알림 설정** — 목표가 도달 시 푸시/이메일 알림
- [ ] **포트폴리오 트래킹** — 보유 종목 수익률 계산

## 6. 기술 스택

> 처음 보는 이름들이 많더라도 괜찮아요. 각 항목이 왜 필요한지 설명합니다.

### 6-1. 프론트엔드 (화면)

| 기술 | 역할 | 선택 이유 |
|---|---|---|
| **Next.js 14** (App Router) | 전체 웹앱 프레임워크 | 페이지 이동이 빠르고, SEO에 유리. 백엔드 API도 같이 만들 수 있음 |
| **TypeScript** | 프로그래밍 언어 | JavaScript에 타입 검사를 추가 → 버그 조기 발견 |
| **Tailwind CSS** | 스타일링 | 클래스 이름으로 빠르게 디자인. design.md 토큰과 연동 |
| **shadcn/ui** | UI 컴포넌트 | 버튼, 테이블, 모달 등 미리 만들어진 컴포넌트. Tailwind 기반 |
| **Recharts** | 차트 라이브러리 | 주가 라인 차트, 거래량 바 차트, 도넛 차트 등 구현 |
| **TanStack Query** | 데이터 불러오기/캐싱 | API 호출 결과를 캐싱해서 속도 향상, 로딩/에러 상태 자동 관리 |
| **Zustand** | 전역 상태 관리 | 검색어, 즐겨찾기 등 여러 페이지에서 공유하는 데이터 관리 |
| **Framer Motion** | 애니메이션 | 카드 진입, 차트 draw, 숫자 카운트업 등 모션 구현 |

### 6-2. 백엔드 (데이터 처리)

| 기술 | 역할 | 선택 이유 |
|---|---|---|
| **Next.js API Routes** | 서버 API | 별도 서버 없이 Next.js 안에서 API 엔드포인트 작성 가능 |
| **Supabase** | 데이터베이스 + 인증 | PostgreSQL DB + 로그인/회원가입 기능을 쉽게 제공. 무료 플랜 있음 |
| **Prisma** | DB 쿼리 도구 | TypeScript로 DB를 안전하게 조작하는 도구 |

### 6-3. 외부 데이터 소스 (API)

| 데이터 | 소스 | 비고 |
|---|---|---|
| 국내 주가·시세 | **한국투자증권 Open API** 또는 **KIS Developers** | 무료, 실시간 가능 |
| 글로벌 지수·해외 주가 | **Yahoo Finance (비공식)** 또는 **Alpha Vantage** | Alpha Vantage 무료 플랜 있음 |
| 국내 주식 뉴스 | **네이버 금융 RSS** 또는 **NewsAPI** | RSS는 무료 |
| 수출입 데이터 | **KITA 무역통계 API** (한국무역협회) | 공공 API, 무료 |
| 컨센서스 | **에프앤가이드 API** 또는 **KRX 공시 데이터** | 에프앤가이드는 유료일 수 있음 |

### 6-4. 배포 (서비스 운영)

| 기술 | 역할 | 선택 이유 |
|---|---|---|
| **Vercel** | 웹앱 배포 | Next.js 만든 회사. 코드 푸시하면 자동 배포. 무료 플랜 있음 |
| **GitHub** | 코드 저장소 | 버전 관리, Vercel과 자동 연동 |

## 7. 개발 로드맵

```
Phase 1 — MVP (핵심 기능)
  ├── 프로젝트 셋업 (Next.js + Tailwind + shadcn/ui)
  ├── design.md → CSS 변수 / Tailwind 설정 적용
  ├── 레이아웃 (Sidebar + Header + 페이지 그리드)
  ├── 홈 대시보드 (KOSPI/KOSDAQ 지수 카드)
  ├── 종목 검색 + 자동완성
  ├── 종목 상세 페이지 (차트 + 기본 정보)
  ├── 뉴스 피드
  └── 컨센서스 섹션

Phase 2 — 확장
  ├── 글로벌 시장 페이지
  ├── 수출입 데이터 페이지
  ├── 섹터 히트맵
  └── 즐겨찾기 (Supabase 연동 + 로그인)

Phase 3 — 고도화
  ├── AI 뉴스 요약
  ├── 알림 기능
  └── 포트폴리오 트래킹
```

## 9. 제약 조건 및 고려사항

| 항목 | 내용 |
|---|---|
| **API 비용** | 초기에는 무료 API 위주로 구성. 컨센서스 데이터는 유료 가능성 있음 |
| **실시간 데이터** | 완전한 실시간(웹소켓)은 Phase 2 이후 고려. MVP는 주기적 갱신(polling)으로 구현 |
| **모바일 대응** | 반응형 필수. 직장인 투자자는 모바일로 접속 빈도 높음 |
| **데이터 정확성** | 투자 판단에 영향을 주므로 출처 표시 필수 |
